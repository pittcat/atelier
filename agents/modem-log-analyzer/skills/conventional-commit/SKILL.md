# Conventional Commit (Atelier)

## Format

```
<type>(<scope>): <subject>

<body>

<footer>
```

## Types (required one of)

| Type     | Use |
|----------|-----|
| `feat`     | new user-facing capability |
| `fix`      | bug fix |
| `refactor` | code change without behavior change |
| `test`     | tests only |
| `docs`     | docs only |
| `chore`    | tooling / deps / non-runtime |
| `perf`     | perf improvement |

## Rules

- Subject ≤ 72 chars, imperative, no period.
- Body: explain **why**, not what.
- One commit = one concern.
- Never include AI toolchain output that talks about itself (no "Generated by …").
- Sign-off optional. **DCO not enforced in atelier.**

## Anti-patterns (auto-reject)

- `update stuff`
- `wip`
- `+ refactor + fix` mix (split into two commits).
- Any commit touching `langgraph.json` without also updating `README.md`.

## Commit message body template

```
Why: <one-line rationale>
How: <short summary of approach>
Risk: <low|med|high> + mitigation
Test: <which make target>
```
